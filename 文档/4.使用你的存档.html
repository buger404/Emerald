EMERALD




CREATE YOUR SAVE

游戏的存档由 ESave 进行管理，默认是不创建的，如果需要创建，请转到游戏窗口的Form_Load过程，将注释有“创建存档”的部分的注释符号删去。
存档使用ESave.Create进行创建。

ESave.Create 参数解说
参数名 类型 说明
name String 存档名称，请不要选择太过于常见的名字，防止和其他游戏的存档发生冲突。
[optional] BMKey String 验证存档使用的秘钥，请在代码中妥善做好秘钥的保密，防止秘钥被窃取。
如果你想不出该使用什么秘钥，可以在立即面板中输入Debug.Print GetBMKey来获取秘钥。
一旦设置后，请不要修改你的秘钥，否则将导致游戏存档完全丢失。

每次创建后，都将会读取存档，读取时将会进行存档检查，非法修改的数据将会被删除。
即使对方使用Emerald强制读取你的存档，也不会有任何问题。您发布的游戏中的Emerald的存档检测功能依然健在，除非对方获取您的源代码，否则将不能实现修改您的游戏存档并在您的游戏中正常运行的目的。

USING SAVE

你可以使用 ESave.GetData 获取存档，并使用 ESave.PutData 设置存档数据。

ESave.GetData 参数解说
参数名 类型 说明
name String 数据名称


NOTE：这是一个函数。当指定的存档数据不存在时将返回空字符串。

ESave.PutData 参数解说
参数名 类型 说明
name String 数据名称
value String 数据值


NOTE：每次数据写入后，存档都将会自动保存，请不要频繁修改存档数据。
